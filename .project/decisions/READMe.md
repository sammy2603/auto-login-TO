# Architecture Decision Records (ADR)

Esta pasta contém todas as decisões arquiteturais do projeto.

## Objetivo

Registrar o contexto, as alternativas consideradas, a decisão tomada e suas consequências.

## Convenção

Cada ADR deve seguir o padrão:

ADR-0001-nome-da-decisao.md

ADR-0002-outra-decisao.md

...

## Estrutura

- Contexto
- Problema
- Alternativas
- Decisão
- Consequências

Os ADRs nunca devem ser apagados.

Caso uma decisão seja alterada, um novo ADR deverá ser criado referenciando o anterior.
