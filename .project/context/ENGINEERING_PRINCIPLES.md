# Engineering Principles

## 1. Produto antes da tecnologia

As decisões devem priorizar o valor para o usuário, e não a tecnologia utilizada.

---

## 2. Arquitetura antes da implementação

Nenhuma funcionalidade será implementada sem uma definição clara de onde ela pertence.

---

## 3. Responsabilidade única

Cada módulo deve possuir uma única responsabilidade.

---

## 4. Código legível

Código é escrito para pessoas.

O compilador é apenas um detalhe.

---

## 5. Documentação faz parte do software

Código e documentação devem evoluir juntos.

---

## 6. Refatoração é desenvolvimento

Melhorar código existente faz parte do trabalho.

---

## 7. Simplicidade vence complexidade

Sempre escolher a solução mais simples que resolva corretamente o problema.

---

## 8. Evitar acoplamento

Bibliotecas externas nunca devem dominar a arquitetura.

---

## 9. Testabilidade

Todo componente deve ser projetado para facilitar testes.

---

## 10. Evolução contínua

O software deve ser preparado para crescer durante anos.
